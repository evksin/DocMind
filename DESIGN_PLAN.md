# DocMind Design Plan — пошаговая реализация DESIGN_SPEC.md

Цель: привести UI к стилю premium SaaS (Notion, Linear, Stripe), используя Tailwind CSS и только HTML.

---

## Шаг 1: Подключить Tailwind CSS

- Добавить Tailwind в проект (CDN или build).
- Для простоты MVP — использовать Tailwind CDN в `<head>` всех страниц.
- Убедиться, что стили не конфликтуют с существующим `styles.css` (при необходимости переименовать/отключить старые стили после миграции).

---

## Шаг 2: Базовый layout и общие компоненты

- Создать общий визуальный каркас: шапка (логотип/название), основной контейнер с отступами.
- Задать фон `gray-50`, типографику (headings `gray-900`, body `gray-600`).
- Ввести классы-хелперы для карточек (белый фон, тень, скругления), основных и вторичных кнопок.
- Сохранить одну точку входа стилей (Tailwind + при необходимости минимальный custom CSS).

---

## Шаг 3: Экран Login

- Переверстать `index.html` под Tailwind.
- Карточка по центру, форма «Имя пользователя» + кнопка «Войти».
- Primary button (blue-500), пустые состояния и сообщение об ошибке в стиле карточки.
- Без лишних элементов, спокойный минималистичный вид.

---

## Шаг 4: Экран Dashboard

- Переверстать `dashboard.html`.
- Список документов в виде карточек или таблицы (по спецификации — tables).
- Кнопки «Анализировать» и «Удалить» — вторичный стиль/ссылки.
- Ссылка «Загрузить документ» — primary.
- Пустое состояние: аккуратная карточка с текстом «Пока нет документов» и призывом к загрузке.

---

## Шаг 5: Экран Upload Document

- Переверстать `upload.html`.
- Область загрузки файла (file upload area): крупная, с рамкой/пунктиром, иконкой или текстом «Выберите файл».
- Select типа анализа и кнопка «Анализировать» в одной карточке.
- Состояние загрузки/анализа: индикатор (спиннер или текст «Загрузка…» / «Анализ…»).

---

## Шаг 6: Экран Choose Analysis Type

- Переверстать `analyze.html` (выбор типа анализа для уже загруженного документа).
- Показать имя документа, select типа анализа, primary кнопку «Анализировать».
- Единый стиль с остальными экранами (карточки, отступы, кнопки).

---

## Шаг 7: Экран Analysis Result

- Переверстать `result.html`.
- Результат в «result box»: карточка с контентом, читаемая типографика (pre-wrap для переносов).
- Кнопка/ссылка «Назад к дашборду» — вторичный стиль.
- Состояние ошибки — в том же стиле карточки, без резких цветов.

---

## Шаг 8: Финальная согласованность и полировка

- Пройти по всем экранам: единые отступы, размеры кнопок, скругления карточек.
- Проверить пустые и ошибочные состояния на всех страницах.
- Убедиться, что вся существующая функциональность сохранена (логин, загрузка, анализ, удаление, навигация).
- При необходимости оставить один минимальный `styles.css` только для кастомных правок, не дублирующих Tailwind.

---

## Порядок выполнения

Выполнять по шагам: **1 → 2 → 3 → 4 → 5 → 6 → 7 → 8**.

После каждого шага проверять: страницы открываются, формы и кнопки работают, API не менялись.
